How it's measured

Two independent judges

The model that decides what the filter removes must not be the model that grades the result — otherwise the filter grades its own output. So the labels live in two separate stores, and the scoring judge never sees the filter's decisions.

① System judge · Sonnet

Drives the filter

Labels every candidate Relevant / Irrelevant. Pipeline 2 removes the Irrelevant ones before reranking. These labels are read only by the filter.

Sonnet → label → filter
② Eval judge · Haiku

Scores the result

A different model independently labels the same candidates and is the only thing that scores P@5 / NDCG — for both pipelines. GPT-swappable via one env var.

Haiku → label → score

Judge quality

How much do the two judges agree?

Cohen's κ between the system judge and the independent eval judge, by type. Overall 0.82 (substantial) — but decade is the weakest (0.50), and it's also where the filter does the most work. The heaviest lifting sits on the shakiest labels.

Label → measure → decide.

The offline half of a production loop: label pairs, find which query types the filter helps, then distill labels into the ranker over time. Labels are cached; at serving time the filter is a dictionary lookup, not a live API call.
